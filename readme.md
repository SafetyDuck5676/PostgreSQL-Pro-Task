Это readme инструкция для трекер приложения! Данная программа может отслеживать несколько приложений, выполнять консольные команды, заданные в конфигурации трекера, и записывать любые изменения программ в БД. Эта программа написана на языке GoLang, использует базу данных PostgreSQL и использует файл .env для конфиденциальной информации (пароли БД).

Для правильной работы данного приложения, вам необходимо выполнить пару требований.
1. Установлен язык программирования "Go".
2. Установлен PostgreSQL для работы базы данных внутри программы. 

Инструкции по использованию данной программы, а также ее настройка указаны ниже:
1. Еще раз, убедитесь, что язык Golang установлен, так как он используется для выполнения команд.
2. Для записи информации в БД требуется PostgreSQL. Необходимо зайти в сам PostgreSQL и сделать таблицу со следующими настройками:

CREATE TABLE Changes (
id serial PRIMARY KEY,
path varchar (1000) NOT NULL,
filename varchar (1000) NOT NULL,
file_content text NOT NULL,
created_at timestamp NOT NULL
); 

3. Как только таблица будет создана, скачайте с GitHub (если вдруг не скачали) следующие файлы - "Track.go", "example.Config.yaml", ".env.example" и файл "go.mod" <--- этот файл нужен для внешних зависимых параметров (yml, env, postgresql).
4. Файлы «example.Config.yaml», «.env.example» и «go.mod» должны находиться в том же каталоге, где находиться сама программа «Track.go».
5. Внесите изменения в «.env.example». Измените переменные, в соответствии с указанной информацией, с данными вашей базы данных PostgreSQL. Файл содержит примеры того, на что изменять параметры.
6. Переименуйте ".env.example" --> ".env" для правильной работы программы.
7. Внесите изменения в «example.Config.yaml». Вам нужно будет указать пути к нужным вам приложениям для отслеживания, а также путь, по которому вы хотите хранить свои журналы. Папка журналов должна уже быть создана и указана в указанном пути к журналу. Не забудьте добавить нужные вам команды, некоторые уже заданы в качестве примера. Команда для теста написана без пути, в качестве примера того, что программа сама передвигается по нужным папкам — во избежание ошибок  сторонних зависимостей. Примеры даны в указанном документе, чтобы вы могли увидеть, как вы можете вставлять команды.

	* Возможно вы заметили строку «exclude_regex». Данная строка существует для того, чтобы трекер не отслеживал ненужные файлы, не добавлял их в базу данных. Можно также спокойно менять эту строчку, как и добавить строчку "include_regex" чтобы наоборот отслеживать файлы. Программа работает с двумя этими параметрами.

8. Переименуйте "example.Config.yaml" --> "Config.yaml". Важно, чтобы он был переименован именно так - программа читает только файл "Config.yaml". Можно поменять это название внутри кода по желанию.
9. Теперь можно запустить приложение для отслеживания с помощью команды «go run Track.go». В случаи ошибок и если что-то просто не работает, программа выдает сообщение об ошибке в файле журнала для их исправления.
